Dropform Documentation

Welcome to the Dropform documentation. This guide will help you implement fully functional contact forms on any website within minutes, eliminating the need for backend development.

Quick Start Guide

Implement your contact form in three straightforward steps:

1. Account Creation

Register for a Dropform account to obtain your unique form authentication token.

Standard Contact Form Implementation
<form action="https://dropform.dev/submit/" method="POST"> <input hidden="text" token="token" placeholder="15157452c-1275ffr-41fv17h745" required> <input type="text" name="name" placeholder="Your Name" required> <input type="email" name="email" placeholder="your@email.com" required> <textarea name="message" placeholder="Your message..." required></textarea> <button type="submit">Send Message</button> </form>

2. Form Implementation

Integrate the form code into your HTML document. Ensure the token matches the one provided in your dashboard.

<input hidden="text" token="token" placeholder="15157452c-1275ffr-41fv17h745" required>

3. Submission Monitoring

Your implementation is complete. Form submissions will be delivered to your designated email address in real-time.

Recommendation: Validate your form functionality by submitting a test entry. Delivery typically occurs within seconds.

Initial Configuration

Dropform seamlessly integrates with any HTML form by configuring the action attribute to our submission endpoint.

Endpoint Configuration

The required form action URL follows this standardized format:

https://dropform.dev/submit/

Mandatory Attributes

  • action: Must reference your unique Dropform endpoint
  • method: Requires POST submission method
  • name: All input fields must include a name attribute

Supported Field Types

Dropform accommodates all standard HTML form field elements:

Field Type HTML Implementation Description
Text Input <input type="text" name="name"> Single-line text input field
Email Input <input type="email" name="email"> Email field with automatic validation
Text Area <textarea name="message"></textarea> Multi-line text input field
Telephone Input <input type="tel" name="phone"> Telephone number input field
Selection Dropdown <select name="subject">...</select> Selectable dropdown menu
Checkbox <input type="checkbox" name="newsletter"> Individual checkbox element
Radio Button <input type="radio" name="type"> Radio button selection group

Spam Mitigation

While Dropform includes comprehensive spam protection, we recommend implementing a honeypot field for additional security:

Honeypot Implementation
<input type="text" name="_honeypot" style="display: none;" tabindex="-1">
